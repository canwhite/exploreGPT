# exploreGPT - 纯Python实现的GPT模型 + TTT

纯Python实现的GPT模型，支持 **Test Time Training (TTT)** 推理模式。

## 快速开始

```bash
python main.py
```

首次运行会自动从 GitHub 下载 `input.txt` 数据集（人名列表）。

## 数据下载

如果本地没有 `input.txt`，程序会自动下载：

```python
if not os.path.exists('input.txt'):
    import urllib.request
    names_url = 'https://raw.githubusercontent.com/karpathy/makemore/988aa59/names.txt'
    urllib.request.urlretrieve(names_url, 'input.txt')
```

数据来源：[karpathy/makemore](https://github.com/karpathy/makemore) 项目的 `names.txt`

## 核心数据流

```
输入: "anna"
  ↓ Tokenizer

  ↓ Embedding (wte + wpe)
  [向量1, 向量2, 向量3, ...]
  ↓ GPT Layers
  ┌──────────────────────────────┐
  │ Layer 0:                     │
  │  1. RMSNorm                  │
  │  2. Multi-Head Attention     │
  │     - Q, K, V 投影           │
  │     - 注意力分数计算          │
  │     - 加权求和               │
  │  3. 残差连接                 │
  │  4. RMSNorm                  │
  │  5. MLP (ReLU)               │
  │  6. 残差连接                 │
  └──────────────────────────────┘
  ↓ Linear (lm_head)

  ↓ Softmax

  ↓

```

## 关键技术点

1. **自动微分 (Value类)**: 无需手动求导，自动构建计算图并反向传播梯度
2. **自注意力机制**: 模型能看到之前的所有字符，学习字符间的依赖关系
3. **位置编码**: 让模型理解字符的顺序信息
4. **残差连接**: 梯度可以更顺畅地流过深层网络（详见[残差连接详解](residual_connection_explanation.md)）
5. **温度采样**: 控制生成文本的随机性和创造性
6. **Test Time Training (TTT)**: 推理时继续训练，让模型动态适应输入上下文

---

## Test Time Training (TTT) 模式

TTT 是本项目的核心创新：**在推理阶段也进行训练**。

### 核心思想

传统推理：
```
预训练 → 冻结参数 → 推理生成
```

TTT 推理：
```
预训练 → 推理时继续训练 → 动态调整 → 生成
```

### TTT 流程图

```
┌─────────────────────────────────────────────────────────────────┐
│  Phase 2: TTT 推理阶段                                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────┐                                            │
│  │ 保存参数快照     │  ← save_params()                           │
│  │ (pretrained)    │                                            │
│  └────────┬────────┘                                            │
│           ↓                                                      │
│  ┌─────────────────┐     ┌─────────────────┐                    │
│  │ 输入上下文       │────→│ TTT 训练循环     │                    │
│  │ [BOS, ...]      │     │ (5步 Adam更新)   │                    │
│  └─────────────────┘     └────────┬────────┘                    │
│                                   ↓                              │
│                          ┌─────────────────┐                    │
│                          │ 生成新 token    │                    │
│                          │ (使用调整后参数) │                    │
│                          └────────┬────────┘                    │
│                                   ↓                              │
│                          ┌─────────────────┐                    │
│  ┌─────────────────┐     │ 输出生成结果     │                    │
│  │ 恢复参数快照     │←────│ "anna"          │                    │
│  │ (独立样本隔离)   │     └─────────────────┘                    │
│  └─────────────────┘                                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### TTT 配置参数

```python
ttt_learning_rate = 0.001  # 更小的学习率，避免过度偏离预训练
ttt_steps = 5              # 每个样本的TTT训练步数
temperature = 0.5          # 生成温度
```

### TTT 的优势

1. **上下文适应**: 模型根据当前输入动态调整，而非固定不变
2. **独立样本隔离**: 每个样本独立进行TTT，恢复快照确保不互相干扰
3. **轻量开销**: 仅需 5 步梯度更新，几乎不增加推理时间

### 两种 TTT 生成模式

| 模式 | 上下文 | 说明 |
|------|--------|------|
| **标准 TTT** | `[BOS, BOS]` | 从空白开始生成，类似标准推理 |
| **进阶 TTT** | `prefix_tokens` | 使用前缀上下文，生成后续内容 |

将代码分为 **5 个核心系统** 来理解：

---

### 1. 神经元：`Value` 类 (自动求导引擎)

这是代码最开始的那一大段。在深度学习中，模型需要知道“我离正确答案差多少”以及“我该往哪边改”。

- **`data`**: 存储当前的数值（比如 0.5）。
- **`grad`**: 存储梯度（改进方向）。初始是 0，通过 `backward()` 计算出来。
- **数学运算 (`__add__`, `__mul__` 等)**：它重载了 Python 的加减乘除。当你写 `a + b` 时，它不仅算出结果，还悄悄记下了“我是由 a 和 b 加出来的”。
- **`backward()`**: 像倒带一样。从最后的错误（Loss）开始，利用**链式法则**把改进意见传回给每一个参数。

---

### 2. 词典：`Tokenizer` (数据预处理)

模型不认识字符，它只认识数字。

```python
uchars = sorted(set(''.join(docs))) # 找出名字里所有的字母：a, b, c...
BOS = len(uchars) # 定义一个特殊数字，代表“开始/结束”

```

- **作用**：它把人名 `"ana"` 变成序列 `[BOS, 0, 13, 0, BOS]`。
- **`vocab_size`**: 你的字典里有多少个“词”（字母 + BOS）。

---

### 3. 大脑结构：`gpt` 函数 (Transformer 架构)

这是模型的核心逻辑，它决定了模型如何“思考”。

- **Embedding (wte & wpe)**：
- `wte` (Token Embedding)：给每个字母一个 16 维的特征向量。
- `wpe` (Position Embedding)：给每个位置（第 1 个字、第 2 个字...）一个特征向量。因为 GPT 需要知道字符的**顺序**。

- **RMSNorm (归一化)**：
- 把数值稳定在一定范围内，防止计算过程中数字变得太大或太小，导致模型“炸”掉。

- **Attention (注意力机制)**：
- 这是最神奇的地方。它计算当前字母与之前所有字母的相关性（`attn_weights`）。
- 通过 `q` (查询)、`k` (键)、`v` (值) 的点积运算，模型能学到：如果前面是 "A"，后面接 "l" 的概率很大。
- 详见 [Linear 函数详解](linear_function_explanation.md) 和 [线性投影详解](linear_projection_explanation.md)

---

### 4. 学习过程：`Training Loop` (训练循环)

这就是模型“挑灯夜读”的过程。

```python
for step in range(num_steps):
    # 1. 前向传播：把名字塞进去，看看模型预测下一个字母是什么
    logits = gpt(token_id, pos_id, keys, values)

    # 2. 计算损失：预测值与真实字母的差距
    loss_t = -probs[target_id].log()

    # 3. 反向传播：计算所有参数的梯度
    loss.backward()

    # 4. 优化：Adam 算法根据梯度，微调参数
    p.data -= lr_t * m_hat / (v_hat ** 0.5 + eps_adam)

```

- **Adam 优化器**：它比普通的调整更聪明，会记录每个参数调整的“惯性”（Momentum），让学习更平滑。

---

### 5. 成果展示：`Inference` (推理)

模型练成了，现在让它写名字。

- **循环生成**：先给它一个 `BOS`。
- **概率采样 (`random.choices`)**：模型不会只选概率最大的那个字母（那样太死板），而是根据概率随机选。
- **Temperature (温度)**：
- 如果你把 `temperature` 调到 **0.1**，它会非常保守，只写最常见的名字（比如 "Anna"）。
- 如果你调到 **1.0**，它会非常狂野，写出一些奇怪的组合（比如 "Xyza"）。

---

### 这个代码的运行流程总结：

1. **初始化**：随机生成一堆数字作为模型的“初始记忆”。
2. **训练**：读 1000 次人名，每次读完都用 `Value` 类的反向传播功能改进一下记忆。
3. **预测**：从空白开始，一个字母接一个字母地往后猜，直到猜到 `BOS` 结束。
